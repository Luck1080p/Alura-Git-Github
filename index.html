VsCode online

  • Após criarmos o arquivo, vamos na opção "source control" adicionamos um comentario e damos um ok. A alteração irá subir no repositorio.
  • Caso a extensão do github esteja instalada no VsCode local, podemos efetuar o procedimento da mesma forma localmente
  • O git precisa estar instalado, e com as variaveis de ambiente configuradas para funcionar localmente

VsCode e Git Local

  • Quando instalamos o git damos um git init para poder logar no repositorio responsalve (se é no github, bitbocket entre outros)
  1. Clicamos no clone do git
  2. Vamos até o terminal e escrevemos git clone + url copiada
  3. Colamos a url e damos um enter, isso fara conexão com o repositorio sendo possível fazer alterações e adições

Os comandos abaixo podem ser utilizados no terminal:

  • Git clone + Url - Puxa o repositorio (pasta do projeto em sí)
  • Code . - irá abrir o VsCode
  • Git log - Irá mostrar todo o historico de mudanças
  • Git log --oneline - mostra os commits de forma mais organizada
  • Para cada commit é gerado um id no qual servira para puxarmos posteriormente
  • Git pull - faz atualizações nos arquivos caso tenha havido alterações
  • Git status - mostra as modificações
  • Git commit -m - coloca uma mensagem
  • Git commit "colocamos o arquivo" -m - colocamos a mensagem do commit
  • Git push - "empurra" todos os commits no repositorio para o main
  • Git push only "nome do arquivo"- empurra apenas o arquivo em especifico
  • Git push only main
  • Git restore --source + numero do commit - restaura o codigo na versão anterior
  • Podemo criar uma nova branch (ramificação) além da main, para que a parte de desenvolvimento não atrapalhe a integração continua
  • Git checkout -b "nome do arquivo"
